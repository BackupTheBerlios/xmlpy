  2. Introduction

    XML-based Python is a XML-based syntax for the object-oriented programming language Python for scripting. The idea is to generate XML-based Python programs by XML stylesheet transformations (XSLT and translate the resulting XML document into normal Python syntax for execution. One application for this idea are to tranform a dynamic HTML template like a "Python Server Page" (PSP) into servlet code for an application server for example the Webware Server.

    Look to the following simple example:

      <?xml version = "1.0" encoding = "UTF8"?>
  <!DOCTYPE python SYSTEM "xmlpy.dtd">
  <python>
    <def name = "fac">
      <parameter name = "x" default = "1"/>
      <if>
        <condition expr = "x == 1">
          <return>
            <expr>1</expr>
          </return>
        </condition>
        <else>
          <return>
            <expr>x * fac(x-1)</expr>
          </return>
        </else>
      </if>
    </def>
    <print>
      <expr>fac(3)</expr>
    </print>
  </python>

    This xmlpy document is equivalent to the following Python program:

    def fac(x=1): if x == 1: return 1 else: return x * fac(x-1)

    print fac(3)

    This looks quite simple!

  3. XML scheme

    xmlpy is defined by a documented XML scheme ("xmlpy.xsd"). Additionally there is a DTD ("xmlpy.dtd"). The design principle for the XML scheme is to translate all Python statements to XML elements. Expressions can be written as usual. The base syntax is Python Version 2.1.
